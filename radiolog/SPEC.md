# RadioLog Web App SPEC

## 1. 개요

- 목적: 무선 운용 일지를 웹에서 입력/관리한다.
- 범위: 정적 웹앱(`index.html`, `script.js`, `style.css`)과 간단 서버(`server.ps1`)를 포함한다.
- 서버: `milkit/milkit.psm1`을 사용해 정적 파일을 서빙한다.
- 저장: 현재 작성 중인 일지는 브라우저 `localStorage`에 저장한다.
- 원칙: 루트 `AUDIT.md`의 최소 구현/최소 검증을 따른다.

## 1.1 서버 인터페이스

- `GET /` -> `index.html`
- `GET /index.html`
- `GET /style.css`
- `GET /script.js`
- `GET /health` -> `{ "status": "ok" }`

## 2. 고정 운용 대상

- 상급부대: `1DIV` 1개
- 예하부대: `0FA`, `1FA`, `2FA`, `3FA`, `4FA`
- 사단망: `작전망`, `행정군수망`, `PRE(행정군수망)`
- 여단망: `CF`, `F`, `PRE(위치취합보고)`
- 관등: `이병`, `일병`, `상병`, `병장`
- 감명도: `1/1`, `2/2`, `3/3`

## 3. 일지 템플릿 규칙

선택 날짜마다 아래 고정 양식을 자동 생성한다.

- 사단망:
  - `작전망` + `오전` + `1DIV`
  - `작전망` + `오후` + `1DIV`
  - `행정군수망` + `오전` + `1DIV`
  - `행정군수망` + `오후` + `1DIV`
  - `PRE(행정군수망)` + `08:00/10:00/12:00/14:00/16:00` + `1DIV`
- 여단망:
  - `CF` + `08:00/10:00/12:00/14:00/16:00`(고정) + 각 `FA` 부대
  - `CF` + `직접입력 1/직접입력 2`(시간 미고정) + 각 `FA` 부대
  - `F` + `직접입력`(시간 미고정, 행 공통 1개) + 각 `FA` 부대
  - `PRE(위치취합보고)` + `08:00/10:00/12:00/14:00/16:00` + 각 `FA` 부대

총 행 수는 74건이다.

## 4. 입력 및 완료 판정

일반 교신 행(`사단망 작전/행정군수`, `여단망 CF/F`)은 아래 4개 기본 입력값을 가진다.

- 송신 감명도
- 수신 감명도
- 상대 교신자 관등
- 상대 교신자 성명

PRE 행(`사단망 PRE`, `여단망 PRE`)은 아래 2개 PRE 전용 입력값을 가진다.

- 수신 상태(`성공/실패`)
- 상대 보고자

시간 입력:

- `사단망` 4개 셀은 셀별 `실제시간(HH:MM)`을 입력한다.
- `CF`의 `직접입력 1/2` 행은 행별 공통 시간(HH:MM)을 입력한다.
- `F` 행은 행 공통 시간(HH:MM) 1개를 입력한다.
- `PRE`는 고정 시간 슬롯(08/10/12/14/16) 행을 사용하므로 별도 시간 입력을 받지 않는다.

완료 판정:

- 일반 교신 행의 기본 4개 값은 필수다.
- 시간 입력 대상 일반 교신 행은 `시간`도 필수다.
- PRE 행은 `수신 상태`가 필수다.
- PRE 행에서 `수신 상태=성공`인 경우 `상대 보고자`도 필수다.
- PRE 행에서 `수신 상태=실패`인 경우 `상대 보고자`는 선택이다.
- 필수값 중 하나라도 비어 있으면 미완료다.

## 5. 저장 인터페이스

- 마지막 선택 날짜 키: `radiolog:selectedDate`
- 날짜별 일지 키: `radiolog:journal:<YYYY-MM-DD>`
- 날짜별 작성자 키: `radiolog:author:<YYYY-MM-DD>` (`am`, `pm`)
- 값 형식: 행 배열(JSON 문자열, 각 행은 `recordedTime`, `preReceiveStatus`, `preReporter` 문자열 필드를 포함)
- 새 날짜 진입 시 데이터가 없으면 즉시 템플릿을 생성하고 저장한다.

## 6. UI 규칙

- 보고서형 표 중심 입력 UI를 사용한다.
- 레이블은 헤딩 계층으로 구분한다.
- `h1`: `외부망`, `내부망`
- `h2`: `사단망`, `여단망`
- `h3`: 여단망 하위의 `CF`, `F`, `PRE(위치취합보고)`
- 현재 운용상 외부망에는 사단망만, 내부망에는 여단망만 존재하더라도 화면에서 구분 헤딩은 별도로 표시한다.
- 사단망은 `시간(행) x 망(열)` 교차표를 사용한다.
- 여단망은 `시간(행) x 대대(열)` 매트릭스를 사용한다.
- 사단망 PRE(`PRE(행정군수망)`)는 사단망 우측에 별도 표로 병렬 배치한다.
- 여단망 PRE(`PRE(위치취합보고)`)는 `F` 표 아래 별도 표로 분리 배치한다.
- 일반 교신 셀은 `송신/수신 감명도`, `관등`, `성명` 입력을 가진다(`select` 기반 유지).
- PRE 셀은 `수신 상태`, `상대 보고자` 입력을 가진다.
- 일반 교신 셀 입력 배치는 `송신/수신` 1줄, `관등/성명` 1줄로 유지한다.
- 사단망 셀에는 `실제시간(input[type=time])` 1줄을 추가한다.
- `CF` 직접입력 2개 행과 `F` 행은 행 머리글 셀에서 `시간(input[type=time])`을 입력한다.
- 매트릭스 셀 내부에 카드형 래퍼(별도 박스/패널)는 사용하지 않는다.
- 셀 단위 `최신 수정 시각` 표시는 사용하지 않는다.
- 요약(summary) 영역은 사용하지 않는다.
- 메타 영역(제목/기준일/버튼/작성자)은 단일 플랫 바 구조로 구성하고 별도 카드 섹션으로 분리하지 않는다.
- 작성자 기록은 `오전`, `오후` 2개 입력으로 제공한다.
- 사단망/여단망의 1번 컬럼(행 라벨) 너비는 동일한 고정폭으로 유지한다.
- Desktop 전용 고정 레이아웃을 사용하고, 매트릭스 너비는 화면 폭 안에 맞춘다.
- 반응형 미디어쿼리는 사용하지 않는다.
- 전체 UI는 무채색 Gray 계열만 사용한다.
- 페이지 배경은 매우 연한 중성 회색(`#f5f5f5`)으로 고정한다.
- 강조 요소(버튼, 포커스, 상태 구분)도 Gray 명도 차이로만 표현한다.
- 디자인 톤은 현대적 문서형(예: Notion/Toss/Vercel 스타일)으로 유지한다.
- 카드형 대시보드, 장식성 효과, 복잡한 시각 요소는 제외한다.
