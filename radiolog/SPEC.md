# RadioLog Web App SPEC

## 1. 개요

- 목적: 무선 운용 일지를 웹에서 입력/관리한다.
- 범위: 정적 웹앱(`index.html`, `view.html`, `script.js`, `view.js`, `style.css`, `view.css`)과 간단 서버(`server.ps1`)를 포함한다.
- 서버: `milkit/milkit.psm1`을 사용해 정적 파일을 서빙한다.
- 저장: 현재 작성 중인 일지는 브라우저 `localStorage`에 저장한다.
- 원칙: 루트 `AUDIT.md`의 최소 구현/최소 검증을 따른다.

## 1.1 서버 인터페이스

- `GET /` -> `index.html`
- `GET /index.html`
- `GET /view` -> `view.html`
- `GET /view.html`
- `GET /style.css`
- `GET /view.css`
- `GET /script.js`
- `GET /view.js`
- `GET /health` -> `{ "status": "ok" }`

## 2. 고정 운용 대상

- 상급부대: `1DIV` 1개
- 예하부대: `0FA`, `1FA`, `2FA`, `3FA`, `4FA`
- 사단망: `작전망`, `행정군수망`, `PRE(행정군수망)`
- 여단망: `CF`, `F`, `PRE(위치취합보고)`
- 관등: `이병`, `일병`, `상병`, `병장`
- 감명도: `1/1`, `2/2`, `3/3`

## 3. 일지 템플릿 규칙

선택 날짜마다 아래 고정 양식을 자동 생성한다.

- 사단망:
  - `작전망` + `오전` + `1DIV`
  - `작전망` + `오후` + `1DIV`
  - `행정군수망` + `오전` + `1DIV`
  - `행정군수망` + `오후` + `1DIV`
  - `PRE(행정군수망)` + `08:00/10:00/12:00/14:00/16:00` + `1DIV`
- 여단망:
  - `CF` + `08:00/10:00/12:00/14:00/16:00`(고정) + 각 `FA` 부대
  - `CF` + `불시교신`(시간 미고정) + 각 `FA` 부대
  - `F` + `직접입력`(시간 미고정, 행 공통 1개) + 각 `FA` 부대
  - `PRE(위치취합보고)` + `08:00/10:00/12:00/14:00/16:00` + 각 `FA` 부대

총 행 수는 69건이다.

## 4. 입력 및 완료 판정

일반 교신 행(`사단망 작전/행정군수`, `여단망 CF/F`)은 아래 4개 기본 입력값을 가진다.

- 송신 감명도
- 수신 감명도
- 상대 근무자 관등
- 상대 근무자 성명

일반 교신 행 미교신 입력:

- 일반 교신 셀은 우클릭 `미교신 설정`으로 미교신 상태로 전환할 수 있다.
- 미교신 상태에서는 `미교신 사유`를 선택한다.
- 사유 목록은 `중계소 이상`, `감명도 저조`, `중계소 키물림`, `무전실 폐쇄`, `기타`를 사용한다.
- `기타` 선택 시 별도 입력칸은 사용하지 않고 `비고(특이사항)`를 사용한다.

PRE 행(`사단망 PRE`, `여단망 PRE`)은 아래 3개 PRE 전용 입력값을 가진다.

- 수신 상태(`성공/실패`)
- 근무자 계급
- 근무자 성명

비고 입력:

- 모든 행(일반 교신/PRE 공통)은 선택 입력 `비고(특이사항)` 텍스트를 가진다.

시간 입력:

- `사단망` 4개 셀은 셀별 `실제시간(HH:MM)`을 입력한다.
- `CF`의 `불시교신` 행은 행별 공통 시간(HH:MM)을 입력한다.
- `F` 행은 행 공통 시간(HH:MM) 1개를 입력한다.
- `PRE`는 고정 시간 슬롯(08/10/12/14/16) 행을 사용하므로 별도 시간 입력을 받지 않는다.

완료 판정:

- 일반 교신 행의 기본 4개 값은 필수다.
- 시간 입력 대상 일반 교신 행은 `교신시각`도 필수다.
- PRE 행은 `수신 상태`가 필수다.
- PRE 행에서 `수신 상태=성공`인 경우 `근무자 계급/성명`이 모두 필수다.
- PRE 행에서 `수신 상태=실패`인 경우 `근무자 계급/성명`은 선택이다.
- 일반 교신 행이 미교신 상태인 경우 사유 입력 여부와 무관하게 미완료로 본다.
- `비고`는 완료 판정 필수가 아니다.
- 필수값 중 하나라도 비어 있으면 미완료다.

## 5. 저장 인터페이스

- 마지막 선택 날짜 키: `radiolog:selectedDate`
- 날짜별 일지 키: `radiolog:journal:<YYYY-MM-DD>`
- 날짜별 작성자 키: `radiolog:author:<YYYY-MM-DD>` (`am`, `pm`)
- 값 형식: 행 배열(JSON 문자열, 각 행은 `recordedTime`, `preReceiveStatus`, `counterpartyRank`, `counterpartyName`, `note`, `noContactReason` 문자열 필드와 `isNoContact` 불리언 필드를 포함하고, `preReporter`는 호환용으로 유지)
- 새 날짜 진입 시 데이터가 없으면 즉시 템플릿을 생성하고 저장한다.
- `/view`는 로컬스토리지 저장값을 SOT로 조회만 수행하며, 데이터 자동 생성/저장은 하지 않는다.

## 6. UI 규칙

- 보고서형 표 중심 입력 UI를 사용한다.
- 레이블은 헤딩 계층으로 구분한다.
- `h1`: `외부망`, `내부망`
- `h2`: `사단망`, `여단망`
- `h3`: 여단망 하위의 `CF`, `F`, `PRE(위치취합보고)`
- 현재 운용상 외부망에는 사단망만, 내부망에는 여단망만 존재하더라도 화면에서 구분 헤딩은 별도로 표시한다.
- 사단망은 `시간(행) x 망(열)` 교차표를 사용한다.
- 여단망은 `시간(행) x 대대(열)` 매트릭스를 사용한다.
- 사단망 PRE(`PRE(행정군수망)`)는 사단망 우측에 별도 표로 병렬 배치한다.
- 여단망 PRE(`PRE(위치취합보고)`)는 `F` 표 아래 별도 표로 분리 배치한다.
- 일반 교신 셀은 `송신/수신 감명도`, `관등`, `성명` 입력을 가진다(`select` 기반 유지).
- PRE 셀은 `수신 상태`, `근무자 계급/성명` 입력을 가진다.
- 일반 교신 셀 우클릭 시 커스텀 컨텍스트 메뉴를 열고 `미교신 설정` 또는 `교신으로 전환` 동작을 제공한다.
- 미교신 상태 일반 교신 셀은 블록 스타일로 표시하고, 일반 교신 입력 대신 미교신 사유 입력 UI를 표시한다.
- 미교신 상태 일반 교신 셀 입력 구조는 `미교신` 텍스트 1줄 + `미교신 사유 select` 1줄로 고정한다.
- 미교신 상태 일반 교신 셀에서도 `비고` 버튼은 동일하게 유지한다.
- 각 표의 2번 컬럼은 `구분`으로 고정하고, 안내 정보는 행당 1회만 표시한다.
- 사단망 일반 행 `구분`은 `교신시각/송수신 감명도/근무자`를 세로 스택으로 표시한다.
- 여단망 일반 행(`CF`,`F`) `구분`은 `송수신 감명도/근무자`를 세로 스택으로 표시한다.
- PRE 행 `구분`은 `수신상태/근무자`를 세로 스택으로 표시한다.
- 셀 내부 메타 라벨은 사용하지 않는다.
- 일반 교신 셀 입력 배치는 라벨 없이 `송신/수신` 1줄, `관등/성명` 1줄로 유지하고, 사단망 셀에는 상단 `교신시각(input[type=time])` 1줄을 추가한다.
- 모든 입력 셀(일반 교신/PRE 공통) 우상단에 `비고` 버튼을 표시한다.
- `비고` 버튼은 셀 우상단 고정 `16x16` 원형으로 표시한다.
- `비고` 버튼 hover/focus 시 `비고작성` 툴팁을 표시한다.
- `비고` 버튼 클릭 시 행 단위 팝오버(`textarea`)를 열어 특이사항을 입력한다.
- 팝오버는 한 번에 1개만 열 수 있고, 닫기/바깥 클릭/`Esc`/다른 셀 비고 버튼 클릭 시 닫힌다.
- 비고 내용은 팝오버가 닫히는 시점에 저장한다.
- 비고 내용이 있는 셀은 `비고` 버튼에 점/강조 표시로 상태를 표시한다.
- `구분` 스택과 매트릭스 입력행은 행 수(2칸/3칸) 기준으로 높이를 균등 분할해 하단 여백 없이 표시한다.
- `CF` 불시교신 행과 `F` 행은 행 머리글 셀에서 `시간(input[type=time])`을 입력한다.
- 매트릭스 셀 내부에 카드형 래퍼(별도 박스/패널)는 사용하지 않는다.
- 셀 단위 `최신 수정 시각` 표시는 사용하지 않는다.
- 요약(summary) 영역은 사용하지 않는다.
- 메타 영역은 단일 플랫 바 구조로 구성하고 별도 카드 섹션으로 분리하지 않는다.
- 상단 Header 1행은 중앙 정렬 제목 `무선망 운용일지`를 표시한다.
- 상단 Header 2행은 우측 정렬 세로 컬럼으로 `날짜`, `오전 근무자`, `오후 근무자`를 표시한다.
- 상단 Header 3행은 `초기화`, `보기 모드` 버튼을 표시한다.
- `보기 모드` 버튼은 `/view`를 새 탭으로 연다.
- 사단망/여단망의 1번 컬럼(행 라벨) 너비는 동일한 고정폭으로 유지한다.
- `구분` 컬럼(2번 컬럼)은 표 간 동일한 고정폭을 유지한다.
- Desktop 전용 고정 레이아웃을 사용하고, 매트릭스 너비는 화면 폭 안에 맞춘다.
- 반응형 미디어쿼리는 사용하지 않는다.
- 전체 UI는 무채색 Gray 계열만 사용한다.
- 페이지 배경은 매우 연한 중성 회색(`#f5f5f5`)으로 고정한다.
- 강조 요소(버튼, 포커스, 상태 구분)도 Gray 명도 차이로만 표현한다.
- 디자인 톤은 현대적 문서형(예: Notion/Toss/Vercel 스타일)으로 유지한다.
- 카드형 대시보드, 장식성 효과, 복잡한 시각 요소는 제외한다.
- `/view`는 매트릭스 구조를 유지한 읽기 전용 화면으로 제공한다.
- `/view` 상단 메타(`기준일/작성자`)는 우측 정렬로 표시한다.
- `/view`는 기본 표시를 A4 세로 비율(210x297mm) 기준으로 구성한다.
- `/view`의 `시간`(1번 컬럼)과 `구분`(2번 컬럼)은 각각 `64px` 고정폭으로 표시한다.
- `/view`의 날짜 기준은 `radiolog:selectedDate`를 우선 사용하고, 값이 없으면 오늘 날짜를 사용한다.
- `/view`는 해당 날짜 `radiolog:journal:<YYYY-MM-DD>` 저장값이 있을 때만 표를 렌더링하고, 없으면 데이터 없음 안내를 표시한다.
- `/view`는 전체 69행을 렌더링하며 비어 있는 값은 `-`로 표시한다.
- `/view` 셀 내부는 값만 표시하고, `교신시각/송수신 감명도/근무자/수신상태/비고` 같은 항목 라벨 텍스트는 표시하지 않는다(구분 컬럼 기준으로 해석).
- `/view`에서 셀 내부 시간값(`recordedTime`)은 사단망 일반 셀에만 표시하고, `CF 불시교신`/`F` 행 셀에서는 표시하지 않는다.
- 단, `/view`에서 `비고`는 선택 입력이므로 값이 있을 때만 라인을 표시하고, 값이 없으면 `비고` 라인을 표시하지 않는다.
- `/view`에서 전체 행의 `비고`가 모두 비어 있으면 `구분` 안내의 `비고` 항목도 표시하지 않는다.
- `/view`는 `select/input/textarea`, `비고` 버튼/팝오버, 컨텍스트 메뉴를 표시하지 않는다.
- `/view` 인쇄 시 표는 페이지 중간에서 분리하지 않고 표 단위로 배치한다.
- `/view` 인쇄 시 상단 메타 영역(`기준일/작성자`)은 숨긴다.
- `/view` 인쇄 시 최상단 중앙에 큰 제목 `무선망 운용일지`를 표시한다.
- `/view` 인쇄 시 최상단에 `무선반장` 서명란을 표시한다.
